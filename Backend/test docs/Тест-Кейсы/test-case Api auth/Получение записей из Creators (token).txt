Название: Получение записей из таблицы Creators (корректные данные в запросе с токеном авторизации)

Номер: GC-001

Шаги:
1.Открыть Postman.
2.Выбрать метод запроса: GET.
3.В поле URL ввести: http://localhost:8080/admin/creators.
4.В хедерах запроса добавить следующий заголовок:
Authorization: token
5.Нажать на кнопку "Send" для отправки запроса

Ожидаемый результат:
Статус код ответа 200.
Тело ответа должно содержать JSON-объект с записями из таблицы Creators.



Название: Получение записей из таблицы Creators (корректный запрос без токена авторизации)

Номер: GC-002

Шаги:
1.Открыть Postman.
2.Выбрать метод запроса: GET.
3.В поле URL ввести: http://localhost:8080/admin/creators.
4.В хедерах запроса не передавать токен авторизации.
5.Нажать на кнопку "Send" для отправки запроса

Ожидаемый результат:
Статус код ответа 500
Тело ответа должно быть пустым



Название: Получение записей из таблицы Creators (корректный запрос с некорректным  токеном авторизации)

Номер: GC-003

Шаги:
1.Открыть Postman.
2.Выбрать метод запроса: GET.
3.В поле URL ввести: http://localhost:8080/admin/creators.
4.В хедерах запроса добавить следующий заголовок:
Authorization: "неккоректный token" например (Wmdsdkcsdmcfvsckvsj)
5.Нажать на кнопку "Send" для отправки запроса

Ожидаемый результат:
Статус код ответа 400 (Bad Request).
Тело ответа должно содержать сообщение об ошибке авторизации.
